# 標題副標設計規格 —「城市獵人」金色中文 overlay

> Rise of the Dragon (Dynamix 1990) 繁中化標題畫面 · 致敬主角孟波＝城市獵人盜版梗
> 引擎即時 overlay，**不改原圖**。手法沿用姊妹作 Willy Beamish（同 DGDS 引擎）金色描邊副標。
> 座標一律 **640×400 hi-res `dst` 疊圖空間**（與 willy `drawGlyph` 同一套）。

---

## 0. 目標畫面實測（intro_13.png，640×400）

逐像素掃描既有金字 / emblem 佔位，定出「乾淨石面」的可用區：

| 元素 | 範圍（640×400） | 備註 |
|---|---|---|
| 上方空帶（emblem 三角尖以上） | y ≈ 8–60，x 全幅 | **完全乾淨**，無亮邊、無金字 |
| Emblem 龍頭三角 | 約 y 60–210，中央 x≈230–410 | 深色浮雕，三角尖 ≈ y60 / x320 |
| 金字「Rise of the」 | y ≈ 211–256，x 128–535 | 中上小字 |
| 接縫（兩段之間） | y 257–261，僅 5px 高 | **太薄，塞不下任何字** |
| 金字「Dragon」大字 | y ≈ 262–399，x 128–540 | 底邊壓到 y=399（無底帶） |
| 「Dragon」底部僅剩「g」尾 | y 372–399，x 288–418 | 中央被尾巴佔住 |
| 左空白邊 | x 0–128，整個高度 | 乾淨 |
| 右空白邊 | x 540–639（≈99px） | 乾淨但窄，底部「n」尾微凸 |

**關鍵結論**：ROTD 的底邊被「Dragon」descender 用到 y=399，**沒有 willy 那種底部留白帶**。底部置中（y≈372）會直接壓到「g」尾巴（已用紅框驗證）。可用的大塊乾淨石面是**最上方那條空帶**。

## 1. 位置（推薦方案）

### ✅ 採用：頂部置中（Top-centered banner）

```
4 字 24×24，字距 adv = 28（24 + 4 間距）
run 寬 = (4−1)×28 + 24 = 108 px
置中 x0 = (640 − 108) / 2 = 266      → 字串橫跨 x 266 .. 374
y_top = 20                            → 字串佔 y 20 .. 44
```

- 每字左上角 x：`266, 294, 322, 350`，全部 `y_top = 20`。
- 上下都有餘裕：頂邊距 20px、距 emblem 三角尖（y≈60）還有 16px 淨空 → **不壓龍頭**。
- 水平完全置中、不碰任何金字（最近的「Rise of the」在 y211 以下）。

### 落選方案與理由

| 方案 | 結論 |
|---|---|
| (a) 16–18px 塞底帶 | ✗ 底帶不存在（descender 到 y399），縮字也會壓「g」尾，且 16px 比 willy 24px 掉質感 |
| (b) 「Rise of the」與「Dragon」之間空檔 | ✗ 接縫僅 5px 高，物理上塞不下 24px |
| (c) 靠右下乾淨角 | ✗ 右側淨寬僅 99px、貼邊、不對稱，看起來像漏掉沒排版（已驗證，畫面飄） |
| (d) **頂部置中** | ✅ 唯一的大塊乾淨石面、可放滿尺寸 24px、置中對稱、不擋英文也不擋 emblem |

**與 willy 的對應**：willy 把「威利奇遇記」當作框住英文 logo 的 CJK caption（放底部）。ROTD 底部被佔滿，於是把同一個「framing caption」邏輯**翻到頂部**——副標仍是替整個英文 logo 加註的一行金色中文，質感與手法一致，只是換邊。

## 2. 配色（從 intro_13 金字取樣）

ROTD 的金比 willy 更偏琥珀／橙金（willy 是 255,201,112 偏奶金）。實測「Dragon」金字色階：

| 用途 | RGB | 來源 |
|---|---|---|
| **金色本體（主色）** | **(238, 165, 0)** | 「Dragon」最大宗金色（高光面主調） |
| 高光（可選，畫頂緣 1px 提亮） | (238, 173, 0) | 實測最亮金 |
| 陰影金（可選，畫底緣加深成漸層） | (189, 97, 0) | 實測最暗金 |
| **描邊（engrave outline）** | **(60, 18, 6)** | 取「Dragon」金字左側浮雕暗邊 ((82,24,8)/(65,40,8)) 再壓深，貼近石面冷調 |

- **最低限可實作版**：本體 (238,165,0) + 描邊 (60,18,6) 兩色即可，已驗證可讀。
- **想跟「Dragon」漸層一致**（加分項）：glyph 本體由上而下做兩段插值 —— 上 1/3 用 (238,173,0)、中段 (238,165,0)、下 1/3 用 (205,117,0)；和大字「Dragon」的「亮頂暗底」漸層方向一致。
- 引擎端用 `nearestPalIdx` 從標題當下調色盤挑最近 index，這裡只給目標 RGB。

## 3. 字級與間距

| 參數 | 值 |
|---|---|
| 每字尺寸 | 24 × 24 px（與 willy glyph 同，full size，不縮） |
| 字距 adv | **28 px**（= 24 + 4，與 willy `adv = 24 + 4` 一致） |
| 4 字總寬（run） | **108 px** = 3×28 + 24 |
| 置中 x0 | **266**（=(640−108)/2） |
| 每字 x | 266 / 294 / 322 / 350 |
| y_top | 20 |

- 置中後橫跨 x266–374，距左右邊各 266px / 266px，**遠不出畫面**。
- 上方淨空 20px、距 emblem 三角尖 ≈16px，**不壓龍頭**（已用實機定格驗證 emblem apex 在 y≈60）。

## 4. 描邊樣式

- **8 向偏移描邊，±2 px**（與 willy 相同手法）：對每字先以描邊色畫 8（或填滿 5×5 鄰域）次偏移 `(dx,dy) ∈ {−2,−1,0,1,2}²＼(0,0)`，再畫金色本體覆蓋。
- **石面 vs 深藍底差異補強**：willy 是深藍底（18,22,64），描邊深藍即可融入；ROTD 是**中性灰石面**（≈50,55,65），對比較弱。因此：
  - 描邊**加深、偏暖黑**：用 **(60,18,6)**（暖黑棕）而非 willy 的冷深藍 —— 暖描邊能讓金字從冷灰石面「跳」出來，且與英文金字的暗浮雕同調。
  - 若實機上仍嫌糊，把描邊外圈再加一層 **±2 的純黑 (0,0,0) 半圈**（只畫最外緣，不必 8 向滿圈）增加對比；內圈保留 (60,18,6) 維持暖調。建議**先上 (60,18,6) 單層測試**，多半已足夠（mock 驗證可讀）。

---

## 5. 設計理由（一句）

把「城市獵人」做成**頂部置中的金色描邊 caption**，是在「ROTD 底部已被 Dragon descender 佔滿、沒有 willy 那條底帶」的硬約束下，唯一能放滿尺寸 24px、置中對稱、又不壓英文金字與龍頭 emblem 的乾淨石面；金色取自「Dragon」本身的琥珀色階、暖黑描邊呼應英文浮雕，讓這行致敬孟波＝城市獵人的中文，看起來像原廠就替整個 logo 加的一行漢字標語，而非後貼的字幕。
